01 · 요구사항 (Requirements)

전제: 범용 타깃 / 인쇄 텍스트 중심 OCR / 100% 로컬 + 수동 파일 내보내기 / Flutter.


핵심 사용자 시나리오 (User Scenarios)

시나리오 1 — 고등학생 지민: "찍고 태그 달고 넘어가기"

지민은 영어 모의고사 채점 후 틀린 어법 문제를 오답렌즈로 촬영한다.
앱이 자동으로 인쇄된 지문·문항을 OCR로 텍스트 추출하고, 지민은 #영어 #모의고사 #어법 태그를 톡톡 붙여 저장한다. 전 과정 10초.
저녁에 #어법 태그만 모아 보며 비슷한 실수 패턴을 한눈에 확인한다.


요구: 빠른 촬영 등록, 자동 OCR, 수동 태그 부착, 태그별 묶음 보기.


시나리오 2 — 취준생 현우: "쌓아두고 키워드로 다시 찾기"

현우는 한국사 기출 문제집의 오답을 평소에 사진만 잔뜩 등록해 둔다(태그는 대충 #한국사).
시험 2주 전, "근현대사" 부분만 복습하고 싶다. 검색창에 "갑오개혁" 을 치면 OCR로 뽑힌 텍스트에서 해당 단어가 포함된 오답들이 바로 뜬다.
지하철에서, 데이터 없이도(오프라인) 검색과 열람이 그대로 된다.


요구: OCR 텍스트 전체 검색, 오프라인 검색·열람, 대량 등록 견디는 목록.


시나리오 3 — 어학 학습자 수아: "내 데이터는 내가 들고 다닌다"

수아는 토익 단어·문법 오답을 1년 넘게 모았다. 폰을 바꾸게 되어 데이터를 잃기 싫다.
설정에서 "내보내기" 를 눌러 이미지+텍스트+태그가 담긴 파일을 만들고, 새 폰에서 "가져오기" 로 그대로 복원한다. 서버도 계정도 없이.


요구: 수동 파일 내보내기/가져오기, 데이터 무결성 보존, 계정 불필요.



기능 분류 (MoSCoW)

✅ Must — 없으면 제품이 성립 안 됨


카메라 촬영 / 갤러리 선택으로 오답 등록
온디바이스 OCR로 인쇄 텍스트 추출 (한국어·영어), 첫 모델 외 네트워크 0
추출 텍스트 편집·수정 (오인식 보정)
해시태그 수동 부착 (다중 태그)
태그 기반 분류·필터링 (태그 선택 시 해당 오답만)
오답 데이터(원본 이미지 + 텍스트 + 태그) 로컬 저장, 완전 오프라인
오답 목록 / 상세 보기


🟡 Should — 핵심 가치를 크게 키움


OCR 텍스트 키워드 검색 (전체 텍스트 검색)
수동 내보내기 / 가져오기 (이미지+텍스트+태그 묶음 파일, 백업·기기 이전)
태그 자동 추천 (기존 태그 빈도/간단 규칙 기반, 클라우드 X)
복습 상태 표시 (안 봄 / 복습함 / 완료)
정렬 (최신순 / 태그별 / 미복습 우선)


🔵 Could — 있으면 좋지만 후순위


다크모드 / 테마
OCR 전 이미지 자동 보정 (자동 크롭·기울기·명암)
통계 대시보드 (태그별 오답 수, 복습률)
로컬 알림 기반 복습 리마인더
항목별 손글씨/메모 추가, 즐겨찾기·중요 표시


⛔ Won't — 이번 버전 명시적 제외


클라우드 자동 동기화 / 계정·로그인
서버 기반 OCR·AI 채점·풀이 생성
수학 수식(LaTeX) 구조 인식 (수식은 이미지+태그로 보관)
소셜 공유 / 협업 / 멀티유저
결제·구독·광고
