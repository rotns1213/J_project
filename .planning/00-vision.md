프로젝트

오답렌즈 (OdapLens)


문제 사진을 찍으면 온디바이스 OCR로 텍스트를 뽑아내고, 해시태그로 분류·검색하는 100% 로컬·오프라인 오답노트 앱.



배경 (Problem)


손으로 오답노트를 만드는 건 시간이 오래 걸리고, 사진만 찍어두면 나중에 다시 찾을 수가 없다 (갤러리에 사진 수백 장이 묻힌다).
기존 학습 앱 대부분은 계정·서버·구독을 요구하고, 풀이/오답 같은 민감한 학습 데이터가 클라우드로 올라간다.
시험장·도서관·지하철 등 네트워크가 불안정한 환경에서 자주 쓰이는데, 온라인 의존 앱은 그때 무용지물이다.


비전 (Vision Statement)


"찍기만 하면 검색 가능한 오답노트가 된다 — 서버 없이, 내 폰 안에서."

오답렌즈는 사진 한 장을 "검색되는 학습 자산"으로 바꾼다. 모든 처리는 기기 안에서 끝나며, 데이터는 사용자만 소유한다.



목표 (Goals)


마찰 최소화: 사진 촬영 → 태그 부착 → 저장까지 3탭 이내, 10초 안에 한 건 등록.
다시 찾기: 등록한 오답을 태그 + OCR 텍스트 키워드로 즉시 검색.
완전 오프라인: 첫 모델 다운로드 이후 네트워크 0. 인식·저장·검색 모두 기기 내에서 동작.
데이터 주권: 서버·계정 없음. 사용자가 원할 때 파일로 직접 내보내고 복원할 수 있음.


타깃 사용자 (범용)

중·고등학생(내신·수능), 대학생·취준생(자격증·공시), 어학 학습자를 모두 포괄한다.
공통점은 "인쇄된 교재·문제집의 오답을 모아 반복 학습" 한다는 행동 패턴이다.

핵심 원칙 (Principles)


Offline-first: 기능 설계의 기본값은 오프라인. 온라인은 예외이며, 이 버전엔 없음.
Privacy by design: 학습 데이터는 기기를 떠나지 않는다 (수동 내보내기 제외).
Zero account / Zero server: 로그인·회원가입·서버 비용 없음.
Capture-then-organize: 먼저 빠르게 담고, 분류는 나중에 가볍게.


비전 범위 밖 (Non-Goals · 이번 버전)


클라우드 자동 동기화, 계정/로그인 시스템
서버 기반 OCR·AI 채점·풀이 생성
수학 수식(LaTeX) 구조 인식 — 수식은 이미지 + 태그로 보관 (인쇄 텍스트 인식에 집중)
소셜 공유·협업·결제·광고


플랫폼 결정 (Platform Decision)

선택: Flutter (iOS + Android 단일 코드베이스)

기준FlutterReact Native네이티브(Swift/Kotlin)온디바이스 OCRML Kit 플러그인 성숙, 오프라인 한/영 인식 양호지원 있으나 파편화최상(Vision/ML Kit)이나 2배 작업오프라인 로컬 DBIsar/Drift/Hive 등 강함보통강함개발 속도(1인·소규모)높음높음낮음결론채택대안과함

핵심 근거: 이번 버전은 인쇄 텍스트 OCR이 중심이라, Google ML Kit의 온디바이스 텍스트 인식(google_mlkit_text_recognition)으로 충분하다. 모델 다운로드 후 완전 오프라인 동작 → "100% 로컬" 비전과 정확히 맞고, 단일 코드베이스로 양 플랫폼을 빠르게 커버한다. (수식 OCR이 핵심 요구가 되면 네이티브 또는 별도 엔진 재검토 필요 — 이번 범위 밖.)

성공 지표 (Success Metrics)


오답 1건 등록 평균 소요 시간 ≤ 10초
인쇄 교재 기준 OCR 텍스트 정확도(체감) ≥ 90%
"원하는 오답을 다시 찾는 데" 평균 ≤ 3탭
네트워크 차단(비행기 모드) 상태에서 전 기능 정상 동작
